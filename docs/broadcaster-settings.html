配信者設定 取扱説明書

メインGUIの「監視配信者」画面で、配信者の登録、生成内容、文字起こし、人物タグ、公開先を設定するための説明書です。コマンド操作は必要ありません。

配信者を登録する基本設定使用タグ候補生成する要素AI・投稿・サーバー配信者別プロンプトキャラクター文字起こし保存後の動作

配信者を登録する

  1. メインGUIで「監視配信者」タブを開きます。
  2. 「配信者ID または user URL」に配信者IDかユーザーページURLを入力します。
  3. 必要なら名前を入力します。空欄なら自動取得されます。
  4. 「配信者登録」を押します。
  5. 登録された行を選び、編集画面を開きます。

変更後は画面下部の「保存」を押します。「キャンセル」を押すと変更は保存されません。

文字起こし

エンジン

  • FasterWhisper：通常の文字起こし向け。話者分離が不要な場合に使います。
  • WhisperX：話者分離や詳細な時刻合わせが必要な場合に使います。

初期プロンプト

録音の種類や会話の前提を音声認識へ伝えます。空欄なら「これはニコニコ生放送の録画音声です」が使われます。人物名は使用タグ候補へ登録し、ここには会話全体の説明を入れます。

モデル

tiny、base、small、medium、large-v3から選びます。大きいモデルほど一般に精度は上がりますが、時間とGPUメモリも多く使います。

WhisperXの話者分離

  • 2人の会話なら、話者数の最小と最大を両方2にします。
  • 人数不明なら想定範囲を指定します。
  • Hugging Faceトークンと対応モデルの利用許可が必要です。
  • 話者ラベルは人物名の自動判定ではなく、話者を区別する番号です。

Hotwordsは使用タグ候補の「認識支援文字」から自動生成されます。別の欄へ重複入力する必要はありません。

保存後の動作

  • 監視設定は次回以降の放送に使われます。
  • 生成項目と文字起こし設定は、その配信者の次回処理から使われます。
  • 既に作成済みのHTMLは、設定を保存しただけでは変わりません。
  • 既存HTMLへ反映する場合は、タイムシフトGUIから必要なStepを再実行します。

迷った場合の基本設定

  1. 監視と放送終了後のHTML生成をオンにします。
  2. 個別設定を使う場合だけ「この配信者では個別設定を使う」をオンにします。
  3. 要約、抽象的要約画像、タイムラインHTMLを選びます。
  4. 話者分離が不要ならFasterWhisper、必要ならWhisperXを選びます。
  5. 人物名や固有名詞を使用タグ候補へ登録します。
  6. AIや公開先を使う場合だけAPIキーとサーバー設定を入力します。
  7. 「保存」を押します。
